Système de gestion de centre médical

Conception et modélisation d'une base de données pour la gestion d'un centre de santé

Présentation du projet

Conception et implémentation d'un système de gestion de base de données pour un centre médical, couvrant la gestion des patients, des médecins, des rendez-vous, des consultations et des prescriptions médicales. Le projet a nécessité une analyse approfondie des besoins, la création de modèles conceptuels et logiques, et l'implémentation SQL complète.

Ce projet démontre une compréhension solide des principes de modélisation de données, des formes normales, et de la conception de systèmes d'information complexes. L'accent a été mis sur l'intégrité des données, la performance des requêtes et la sécurité des informations médicales sensibles.

Objectifs du système

1

Gestion des patients

Enregistrement complet des informations patients (identité, coordonnées, historique médical, allergies).

2

Gestion du personnel médical

Base de données des médecins avec spécialités, horaires, disponibilités et qualifications.

3

Système de rendez-vous

Planification et gestion des consultations avec vérification des conflits et rappels automatiques.

4

Dossiers médicaux

Historique complet des consultations, diagnostics, traitements et prescriptions.

Modèle conceptuel (MCD)

Le modèle conceptuel de données a été conçu en suivant la méthode Merise, identifiant les entités principales et leurs relations :

Entités principales

Patient

  • ID_Patient (PK)
  • Nom, Prénom
  • Date de naissance
  • Numéro sécurité sociale
  • Coordonnées
  • Groupe sanguin
  • Allergies

Médecin

  • ID_Medecin (PK)
  • Nom, Prénom
  • Spécialité
  • Numéro RPPS
  • Coordonnées
  • Horaires

Rendez-vous

  • ID_RDV (PK)
  • Date et heure
  • Durée
  • Motif
  • Statut
  • ID_Patient (FK)
  • ID_Medecin (FK)

Consultation

  • ID_Consultation (PK)
  • Date
  • Symptômes
  • Diagnostic
  • Observations
  • ID_RDV (FK)

Prescription

  • ID_Prescription (PK)
  • Date
  • Médicaments
  • Posologie
  • Durée traitement
  • ID_Consultation (FK)

Médicament

  • ID_Medicament (PK)
  • Nom commercial
  • DCI
  • Laboratoire
  • Prix
  • Remboursement

Relations et cardinalités

  • Patient ↔ Rendez-vous: Un patient peut avoir plusieurs rendez-vous (1,N)
  • Médecin ↔ Rendez-vous: Un médecin peut avoir plusieurs rendez-vous (1,N)
  • Rendez-vous ↔ Consultation: Un rendez-vous donne lieu à une consultation (1,1)
  • Consultation ↔ Prescription: Une consultation peut générer plusieurs prescriptions (1,N)
  • Prescription ↔ Médicament: Relation N-M via table association

Modèle logique (MLD)

Traduction du modèle conceptuel en modèle logique relationnel avec respect de la 3ème forme normale (3FN) :

Patient(ID_Patient, Nom, Prenom, DateNaissance, NumSecu, Telephone, Email, Adresse, GroupeSanguin)

Medecin(ID_Medecin, Nom, Prenom, Specialite, NumRPPS, Telephone, Email)

RendezVous(ID_RDV, DateHeure, Duree, Motif, Statut, #ID_Patient, #ID_Medecin)

Consultation(ID_Consultation, DateConsult, Symptomes, Diagnostic, Observations, #ID_RDV)

Prescription(ID_Prescription, DatePrescription, Posologie, Duree, #ID_Consultation)

Medicament(ID_Medicament, NomCommercial, DCI, Laboratoire, Prix, TauxRemboursement)

Prescrire(#ID_Prescription, #ID_Medicament, Quantite, Instructions)

Allergie(ID_Allergie, Nom, Type, Gravite)

PatientAllergie(#ID_Patient, #ID_Allergie, DateDetection)

Implémentation SQL

Scripts de création

Création des tables avec contraintes d'intégrité :

-- Table Patient
CREATE TABLE Patient (
  ID_Patient INT PRIMARY KEY AUTO_INCREMENT,
  Nom VARCHAR(50) NOT NULL,
  Prenom VARCHAR(50) NOT NULL,
  DateNaissance DATE NOT NULL,
  NumSecu VARCHAR(15) UNIQUE NOT NULL,
  Telephone VARCHAR(15),
  Email VARCHAR(100),
  Adresse TEXT,
  GroupeSanguin VARCHAR(5)
);

-- Table Medecin
CREATE TABLE Medecin (
  ID_Medecin INT PRIMARY KEY AUTO_INCREMENT,
  Nom VARCHAR(50) NOT NULL,
  Prenom VARCHAR(50) NOT NULL,
  Specialite VARCHAR(50) NOT NULL,
  NumRPPS VARCHAR(11) UNIQUE NOT NULL,
  Telephone VARCHAR(15),
  Email VARCHAR(100)
);

-- Table RendezVous
CREATE TABLE RendezVous (
  ID_RDV INT PRIMARY KEY AUTO_INCREMENT,
  DateHeure DATETIME NOT NULL,
  Duree INT DEFAULT 30,
  Motif VARCHAR(200),
  Statut ENUM('Prévu', 'Confirmé', 'Annulé', 'Effectué') DEFAULT 'Prévu',
  ID_Patient INT NOT NULL,
  ID_Medecin INT NOT NULL,
  FOREIGN KEY (ID_Patient) REFERENCES Patient(ID_Patient),
  FOREIGN KEY (ID_Medecin) REFERENCES Medecin(ID_Medecin)
);

Requêtes complexes

1. Historique patient

Récupération de l'historique complet d'un patient :

SELECT p.Nom, p.Prenom, c.DateConsult, c.Diagnostic, m.Nom AS Medecin
FROM Patient p
JOIN RendezVous r ON p.ID_Patient = r.ID_Patient
JOIN Consultation c ON r.ID_RDV = c.ID_RDV
JOIN Medecin m ON r.ID_Medecin = m.ID_Medecin
WHERE p.ID_Patient = 1
ORDER BY c.DateConsult DESC;

2. Disponibilités médecin

Vérification des créneaux libres :

SELECT DateHeure, Duree
FROM RendezVous
WHERE ID_Medecin = 5
AND DATE(DateHeure) = '2025-01-25'
AND Statut != 'Annulé';

3. Statistiques consultations

Nombre de consultations par spécialité :

SELECT m.Specialite, COUNT(*) AS NbConsultations
FROM Medecin m
JOIN RendezVous r ON m.ID_Medecin = r.ID_Medecin
JOIN Consultation c ON r.ID_RDV = c.ID_RDV
WHERE YEAR(c.DateConsult) = 2024
GROUP BY m.Specialite
ORDER BY NbConsultations DESC;

4. Prescriptions actives

Liste des traitements en cours :

SELECT p.Nom, p.Prenom, med.NomCommercial, pr.Posologie
FROM Patient p
JOIN RendezVous r ON p.ID_Patient = r.ID_Patient
JOIN Consultation c ON r.ID_RDV = c.ID_RDV
JOIN Prescription pr ON c.ID_Consultation = pr.ID_Consultation
JOIN Prescrire ps ON pr.ID_Prescription = ps.ID_Prescription
JOIN Medicament med ON ps.ID_Medicament = med.ID_Medicament
WHERE DATEDIFF(NOW(), pr.DatePrescription) <= pr.Duree;
